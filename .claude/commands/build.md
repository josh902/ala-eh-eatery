You are helping build or extend the Ala Eh Eatery website. Follow this workflow exactly — do not skip steps.

## Step 1: Clarify Before Coding

Ask clarifying questions until you are **≥95% confident** you fully understand every dimension below. Do NOT assume missing details — if something is unclear, ask. Iterate with follow-up questions as needed.

Dimensions to cover:
- **Website goal** — what is this build/change trying to achieve?
- **Target audience** — who are the users?
- **Design style** — visual tone, mood, references, feel
- **Required sections** — which sections or pages are needed?
- **Functionality** — interactive features, forms, integrations, behavior
- **Animations** — motion style, intensity, specific interactions wanted
- **Branding** — colors, fonts, logo usage, voice/tone constraints
- **User experience expectations** — flow, accessibility, performance priorities

## Step 2: Plan Before Implementing

Once you understand the requirements, produce a written plan:
- Break the work into clear **phases** with specific **todos** per phase
- Identify which existing components, data structures, or utilities can be reused
- Call out any ambiguities or decisions that need to be made
- Present the plan to the user and wait for confirmation before writing any code

## Step 3: Implement One Task at a Time

- Work through the todos in phase order
- Complete and **verify** each task before moving to the next
- Do not move on until the current task is polished — check it thoroughly

## Step 4: Self-Check After Every Change

After each task, review the result against all of these:

- [ ] **Responsiveness** — works correctly on mobile, tablet, and desktop
- [ ] **Alignment** — elements line up as intended
- [ ] **Spacing consistency** — padding/margin follows the existing design rhythm
- [ ] **Animation smoothness** — transitions are fluid, not janky
- [ ] **UI hierarchy** — visual weight and flow guide the eye correctly
- [ ] **Code quality** — no dead code, clear naming, consistent patterns
- [ ] **Bugs** — no broken behavior or edge cases
- [ ] **Accessibility** — semantic HTML, keyboard nav, ARIA where needed
- [ ] **Performance** — no unnecessary re-renders, heavy imports, or layout thrash

Fix any issues found before proceeding to the next task.
